PROJECT PROBLEM STATEMENT

1. Problem
Retail investors and analysts often struggle to make timely, evidence-based stock decisions. Relevant information is distributed across multiple sources, and each source requires a different type of analysis. Market trends, technical indicators, fundamental metrics, and risk controls are typically reviewed separately and then combined manually. This workflow is slow, inconsistent, and vulnerable to subjective bias.

2. Target Users
Primary users are retail investors, finance learners, and analysts who need fast, structured insights for both single-stock analysis and two-stock comparison.

3. User Need
Users need a single system that can accept a natural-language stock query and return a decision-ready output. The output should include market trend, technical signal, fundamental view, and a final recommendation with conviction, risk level, and position size guidance.

4. Specific Problem the AI Agent Will Solve
Given a stock query (for example, "MRF") or a comparison query (for example, "Asian Paints vs MRF"), the AI agent must automatically produce a complete and structured analysis while keeping intermediate findings consistent, traceable, and aligned.

5. Why an Agentic Approach Is Suitable
This problem requires distinct capabilities across stages, including market data retrieval, context enrichment, technical interpretation, fundamental reasoning, and governed final synthesis. A single prompt is not reliable for this full workflow. An agentic architecture is suitable because specialist agents can own each stage and pass structured outputs forward, improving reliability, modularity, and end-to-end decision quality.

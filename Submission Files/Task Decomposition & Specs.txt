Task Decomposition and Specs

Version: 3.0
Date: 2026-04-23
Project: Stock Market Analysis Multi-Agent Pipeline (Groq Edition)

1) Purpose
Define the end-to-end job of the agent system: convert natural-language stock input into a structured, decision-ready investment output (single stock or two-stock comparison) with governance checks, risk controls, and reproducible fallbacks using high-speed Groq inference.

2) Scope
In scope:
- Single-stock analysis
- Two-stock comparison
- 5-agent sequential pipeline (Market, Technical, Fundamental, Macro, Synthesis)
- Quick and deep synthesis modes
- Deterministic safety and fallback logic
- API, CLI, and UI-facing structured outputs

Out of scope:
- Portfolio optimization across many assets
- Order execution or brokerage integration
- Guaranteed price prediction accuracy

3) Global Input Contract
Required runtime inputs:
- Query input: single stock query or two stock phrases
- Mode: quick or deep (defaults to quick)
- Save flag: true/false (optional report persistence)

Required environment:
- GROQ_API_KEY (required)
- GROQ_MODEL (defaults to llama-3.3-70b-versatile)
- TAVILY_API_KEY (optional, context enrichment)

4) Global Output Contract
Single-stock primary output fields:
- ticker, company
- market_report, technical_report, fundamental_report, macro_report
- final_report
- structured_output:
  - verdict (BUY/HOLD/SELL)
  - conviction (1-10)
  - risk (LOW/MEDIUM/HIGH)
  - position_size_pct (float)
  - time_horizon (SHORT/MEDIUM/LONG)
- signals (trend, signal, confidence, fund_view, horizon, macro_rating)
- rubric (criteria scores + total/normalized + grade)
- warnings and override notes

Comparison primary output fields:
- stock_a, stock_b (full single-stock outputs)
- comparison_report
- winner
- confidence
- comparison_meta (winner basis, rubric delta, conviction/risk diagnostics)

5) Task Decomposition

Task T0: Environment and model preflight
Goal:
- Validate Groq connectivity and rate limit settings before running pipeline.
Inputs:
- GROQ_API_KEY, GROQ_MODEL, fallback model list.
Outputs:
- Ready/not-ready status and selected model.
Decision points:
- If GROQ_API_KEY missing: fail fast.
- If configured model unavailable: try fallback models (Llama 3.1, Mixtral).
Handling:
- Validate API key via health check endpoint.
- Return actionable failure message if no usable model is found.

Task T1: Input intake and mode normalization
Goal:
- Normalize user input and synthesis mode.
Inputs:
- Raw query or query list; optional mode.
Outputs:
- Sanitized query list; normalized mode (quick/deep).
Decision points:
- Invalid mode -> force quick.

Task T2: Route to single vs comparison workflow
Goal:
- Decide whether to run one-stock or two-stock pipeline.
Inputs:
- Normalized queries from T1.
Outputs:
- Route: single or multi, plus resolved query pair/list.
Decision points:
- If >= 2 non-empty queries: comparison route.
- If one query: run LLM intent detection for comparison.
- If LLM parse fails: use heuristics (vs, versus, and, comma split).

Task T3: Agent 1 ticker resolution and market snapshot
Goal:
- Resolve natural-language query to ticker and fetch stock snapshot.
Inputs:
- User stock query.
Outputs:
- ticker, company, exchange, confidence, reasoning, stock_data.
Decision points:
- LLM-derived ticker hints available/unavailable.
Handling:
- Try Yahoo resolver candidate-by-candidate until success.
- If all candidates fail: raise resolution error.

Task T4: Agent 1 market context + trend report
Goal:
- Build market report and trend classification.
Inputs:
- ticker/company, stock_data, mode, optional Tavily key.
Outputs:
- market_context, market_report, TREND line.
Decision points:
- Tavily enabled for Agent 1.
Handling:
- If Tavily fails/unavailable: continue with Yahoo data only.
- Deterministic trend fallback based on price vs MA50/MA200.

Task T5: Agent 2 technical analysis
Goal:
- Produce technical report with SIGNAL and CONFIDENCE.
Inputs:
- Agent 1 payload, stock_data, mode.
Outputs:
- technical_context, technical_report, SIGNAL line.
Decision points:
- LLM generation success/failure.
Handling:
- Deterministic technical fallback scoring (MA crossovers, RSI, MACD).

Task T6: Agent 3 fundamental analysis
Goal:
- Produce fundamental report with FUNDAMENTAL VIEW and HORIZON.
Inputs:
- Agent 2 payload, stock_data, mode.
Outputs:
- fundamental_context, fundamental_report, FUNDAMENTAL VIEW line.
Decision points:
- Tavily enabled for Agent 3.
Handling:
- Deterministic fundamental fallback scoring (Growth, Margins, Debt).

Task T7: Agent 4 macro & risk specialist
Goal:
- Evaluate broader economic shifts, regulatory risks, and sector headwinds.
Inputs:
- Agent 3 payload, stock_data, mode.
Outputs:
- macro_context, macro_report, MACO RATING line.
Decision points:
- Tavily enabled for Agent 4.
Handling:
- Research interest rates, inflation, and regulatory changes.
- Deterministic macro rating fallback (STABLE/CAUTION/CRITICAL).

Task T8: Agent 5 signal parsing and data validation
Goal:
- Extract normalized signals from all 4 specialists and validate data.
Inputs:
- Agent 1-4 reports, stock_data.
Outputs:
- Parsed signals, warnings.
Decision points:
- Missing parse lines -> UNKNOWN values + warnings.
Handling:
- Regex-based robust parsing with normalization.

Task T9: Rule engine and governance overrides
Goal:
- Apply deterministic risk and consistency rules to final decision.
Inputs:
- Parsed signals, stock_data.
Outputs:
- Governed structured decision + override notes.
Decision points:
- Hard no-buy rules (e.g., technical SELL + HIGH confidence).
- Bearish regime pressure score (0-5 scale).
Handling:
- Calculate position sizing based on conviction and risk tier.
- Apply conviction caps and risk escalation under bearish pressure.

Task T10: Agent 5 report synthesis
Goal:
- Produce final human-readable report + DECISION_JSON.
Inputs:
- Agent 1-4 reports, parsed signals, stock metrics, mode, rule notes.
Outputs:
- final_report, structured_output.
Decision points:
- Quick mode (single pass) vs Deep mode (critique-revision cycle).
Handling:
- Enforce final verdict/conviction/risk lines.

Task T11: Rubric scoring and quality evaluation
Goal:
- Score report quality based on 5 criteria.
Inputs:
- final_report, structured decision, stock_data.
Outputs:
- Rubric payload (scores, total, grade).
Handling:
- Criteria: Trend Relevance, Sector Fit, Visual-Text Alignment, Quote Quality, Completeness.

Task T12: Two-stock comparison decision
Goal:
- Produce winner and confidence with rubric-delta safety checks.
Inputs:
- stock_a output, stock_b output.
Outputs:
- comparison_report, winner, confidence.
Handling:
- Deterministic winner logic if rubric delta is significant.

Task T13: Delivery and persistence
Goal:
- Return payload and optionally save markdown reports.
Inputs:
- Final payload, save flag.
Outputs:
- JSON payload; optional file in reports/.

6) Failure and Fallback Matrix
- Groq unavailable: stop with explicit preflight message.
- Tavily unavailable: continue without context.
- LLM generation failure: deterministic local fallback reports for all agents.
- Agent 5 synthesis failure: deterministic final recommendation and report.

7) Non-Functional Requirements
- Deterministic safety behavior must always be available.
- Rate limit handling (pacing/cooldown) for Groq free tier.
- Mode behavior:
  - quick = lower latency, single-pass synthesis
  - deep = multi-pass critique and richer context

8) Acceptance Checklist
- 5 agents execute in sequence (1-2-3-4-5).
- Every final report includes structured decision fields.
- Governance notes appear for overrides.
- Comparison always returns exactly one winner.
- Pipeline handles rate limits and service unavailability gracefully.
